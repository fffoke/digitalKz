# ТІЛДЕС 2.0 — AI-собеседник (новый фокус)

Разворот проекта: ядро — **разговорная практика казахского с ИИ голосом**.
Персональные задания на основе интересов пользователя → диалог голосом → оценка.

> MVP, без лишней сложности. Whisper (STT) уже есть. Нужен LLM (диалог + оценка
> + генерация заданий) и опционально TTS (голос ИИ).

---

## 1. Что убираем / оставляем

**Убираем:**
- Лента и посты (Дом) — целиком.
- *(рекомендую)* Директ и Дуэли — для фокуса. Голосовая практика теперь в ядре.
- Систему живых преподавателей / групп / уроков / экзаменов / рангов — пока в сторону
  (модели можно не удалять, просто не используем).

**Оставляем:**
- Аутентификацию (register/login/JWT) — как есть.
- **Обучение** — теперь это AI-собеседник (ядро).
- **Профиль** — пустой, только редактирование (имя, username, био, аватар уже есть).

## 3. Пользовательский флоу

```
Регистрация
   │
   ▼
Первый вход → Онбординг интересов (3 вопроса)
   │  POST /api/me/interests
   ▼
LLM генерит персональные задания
   │  POST /api/tasks/generate
   ▼
Вкладка «Обучение»: список заданий-сценариев
   │  GET /api/tasks
   ▼
Открыл задание → старт сессии диалога
   │  POST /api/tasks/{id}/start
   ▼
ДИАЛОГ ГОЛОСОМ (по кругу):
   ① жмёт центральную кнопку → пишет голос
   ② голосовое появляется как в Telegram (волна, play) — можно переслушать
   ③ отправка → whisper расшифровывает → LLM-собеседник отвечает
      POST /api/sessions/{id}/message  (multipart audio)
   ④ ответ ИИ (текст + опц. голос)
   ▼
Завершение → РЕЗУЛЬТАТ
   POST /api/sessions/{id}/finish
   → { выполнение задания %, качество казахского, разбор ошибок }
```

## 4. Онбординг интересов (первый вход)

3 коротких вопроса (ответы → в профиль обучения):
1. **Зачем учишь казахский?** (работа / переезд / семья / культура / путешествия…)
2. **Что тебе интересно?** (темы: спорт, кино, IT, кулинария, бизнес… — мультивыбор + своё)
3. **Где казахский пригодится?** (на работе, на базаре, в семье, в госуслугах, в путешествии…)

Из этого LLM формирует сценарии заданий: например, для «работа в IT + базар» →
«Объясни коллеге задачу на казахском», «Поторгуйся на базаре».

## 5. Экран диалога (ядро UX)

```
┌───────────────────────────────┐
│  Задание: Закажи кофе в кафе   │  ← сценарий сверху
├───────────────────────────────┤
│                               │
│        🤖 Сәлеметсіз бе!       │  ← сообщение ИИ (текст / голос)
│                               │
│  🎙 ▶ ▁▂▅▇▅▂▁ 0:04   (вы)     │  ← ваше голосовое (Telegram-стиль, играется)
│                               │
│        🤖 Жақсы, не қалайсыз? │
│                               │
├───────────────────────────────┤
│            (  🎙  )            │  ← ОДНА центральная кнопка записи
│         нажмите и говорите     │
└───────────────────────────────┘
```

**Голосовое сообщение (как в Telegram):**
- Запись через `MediaRecorder` (формат webm/opus).
- Сразу показывается баббл: кнопка **play/pause**, **волна** (waveform), **длительность**.
- Пользователь может **переслушать себя** до/после отправки (локальный blob URL).
- Файл грузится на бэк (как аватары → `/uploads`) — чтобы оставалась история.

**Центральная кнопка записи:**
- Одна большая круглая кнопка по центру снизу.
- Тап — старт записи (пульсация + таймер), тап ещё раз — стоп → баббл → отправка.
- На время ответа ИИ — индикатор «ИИ думает…».

**Ответ ИИ:**
- MVP: текстовый баббл (на казахском, в роли собеседника по сценарию).
- Enhancement: голос ИИ через TTS (тоже Telegram-баббл).

## 6. Результат (оценка)

После завершения сессии LLM выдаёт:
- **Выполнение задания** — справился ли с целью сценария (0–100%).
- **Качество казахского** — грамматика, словарный запас, естественность.
- **Разбор** — что сказал хорошо, какие ошибки, как сказать правильнее.
- (опц.) общий балл/прогресс для профиля.

## 7. Endpoints

```
# онбординг интересов
POST   /api/me/interests        { motivation, interests[], contexts[] }
GET    /api/me/interests        → есть ли онбординг + сами ответы

# задания
POST   /api/tasks/generate      LLM создаёт задания из интересов
GET    /api/tasks               список заданий пользователя
GET    /api/tasks/{id}          одно задание

# сессия диалога
POST   /api/tasks/{id}/start    → создаёт сессию, отдаёт первое сообщение ИИ
GET    /api/sessions/{id}       сессия + все реплики (+ результат если завершена)
POST   /api/sessions/{id}/message   multipart audio
                                 → whisper → LLM → { transcript, ai_text, ai_audio?, turn }
POST   /api/sessions/{id}/finish    → LLM-оценка → { task_score, language_score, feedback }
```

## 9. Технологии

| Часть | Чем |
|---|---|
| Распознавание речи (STT) | **faster-whisper** (уже есть), `language="kk"` |
| Диалог + оценка + генерация заданий | **LLM** — нужен ключ (Claude / GPT). Рекомендую Claude |
| Голос ИИ (TTS) | опционально, enhancement |
| Запись на фронте | `MediaRecorder` (webm/opus) |
| Плеер голосовых | волна/waveform (своя отрисовка или мелкая либа) |
| Хранение аудио | `/uploads` (как аватары, StaticFiles) |

⚠️ **Нужен LLM-ключ** (Anthropic/OpenAI) в `.env` бэка — без него диалог и оценка
работают только заглушками.

## 12. Порядок реализации
1. **Онбординг интересов** — форма + `POST /api/me/interests` (хранение).
2. **Генерация заданий** — LLM из интересов → список в «Обучении».
3. **Экран диалога** — центральная кнопка + Telegram-голосовое (запись/плеер).
4. **Пайплайн сообщения** — whisper → LLM-ответ (текст).
5. **Результат** — LLM-оценка задания и языка.
6. **Чистка** — убрать посты/дуэли/директ из навигации и кода.
7. *(enhancement)* голос ИИ (TTS), прогресс в профиле.
